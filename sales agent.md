# AI 营销 Agent 产品思考


## 1. 背景

近期 AI 产品高速发展，目前可见的明显有流量增长的产品形态有两种：

- **对话类**（以豆包为代表）
- **AI 搜**（以 Google AI Overview 为代表）

市场上普遍认为 **2026 年为 AI 变现元年**，但对于 AI 两种形态的具体变现方式大部分比较固化。根据字节提出的"营销 Agent"概念，初步梳理如下。

## 3. 要解决什么问题（自我思考）

### 3.1 核心问题清单

要在 AI 对话 / AI 搜场景下做变现，必须先回答这几个问题：

- **意图识别准确率**：用户的模糊表达如何转成结构化意图？
- **推荐相关性**：在意图识别基础上，候选商品 / 服务的匹配质量如何评估？
- **转化漏斗在多轮对话中的归因**：传统漏斗是单页面的，多轮对话漏斗如何重新建模？
- **用户复购意愿**：单次满意之外，能否形成长期信任与复购？

应对方法是一套 **离线评测（标注集、相关性、安全性）+ 在线评测（A/B、长期留存、客诉率）** 的组合拳（详见第 8 章评估体系）。


### 3.3 现状的根本痛点

- 目前购物流程是**割裂的**：搜索、决策、对比、下单、售后跨越多个产品 / 多个 App，AI 对话是天然有机会做"统一接入层"的载体。

---
### 3.4 产品模式对比
| 维度 | Feed 广告 | 搜索广告 | 豆包 AI 导购广告 |
|---|---|---|---|
| **竞价触发** | 流量驱动（每屏自动触发） | **关键词驱动**（用户输入 query 触发） | **意图驱动**（AI 识别到商业意图才触发） |
| **意图信息密度** | 低（兴趣标签，被动推断） | 中（关键词 / 短语，主动但维度有限） | **高**（自然语言多维度约束：场景 + 偏好 + 限制） |
| **库存供给形态** | 固定广告位（每 N 屏一个） | SERP 顶部 / 侧边固定位置 | **动态——AI 决定是否露出 + 露出几个** |
| **广告位结构** | 独立隔离位（与内容分开） | 独立隔离位（带 "Sponsored" 标签） | **嵌入式**（商品卡融合在 AI 回答正文里） |
| **计费点上限** | 商家回传 a / p 不可控 | c 为主，s 仅电商搜索部分可达 | **CPS 可达（抖音电商支付闭环全链路可观测）** |
| **考核点** | LTV | LTV（单 query 价值高，强转化意图） | **LTV × 用户信任分**（双目标） |
| **触点模型** | 单次曝光 - 单次转化 | 单次 query - 单次曝光 - 单次转化 | **多轮对话 + 多商品提及 + MTA 归因** |
| **用户决策状态** | 发现 / 激情驱动 | 需求明确，主动寻找 | 需求复杂模糊，需要 AI 辅助决策 |
| **AI 能力的角色** | AI 主要做素材生成（即创 / Symphony） | AI 重塑结果页（Google AIO） | **AI 本身就是产品** |

## 4. 营销 Agent 的概念

> 不确定目前字节营销 Agent 落地程度如何，以下为个人粗略思考。

### 4.1 产品能力

可以实现在多轮对话中进行 **"服务"的推荐** 和 **"品牌"的露出**，尤其是可以丝滑地在多轮对话中**逐步承接用户后端需求**。

### 4.2 变现能力（S 曲线）

| 阶段 | 判断 | 原因 |
|------|------|------|
| 初期 | 低 | 流量池小（对比 Feed 时长可能短）；曝光机会少（对比搜索，不是单屏幕 5 次）；广告主接受度初期低；技术基建需要时间；用户使用习惯需要培养 |
| 后期 | 高 | 减少交互摩擦，转化率可提升；长尾需求承接；数据量大；预算侧粘度提升；**单次会话的信息密度更高**——后期变现高不只是"流量大了 + 习惯养成了"，更本质的是单次会话的信息密度更高。Feed 广告一次曝光大概 3-5 秒、信息维度 2-3 个；AI 对话一轮交互可以承载用户**意图 + 约束 + 偏好**等 10+ 维度信息。这种用户主动暴露的高密度意图数据，是 Feed 时代从未有过的资产。变现潜力的真正来源是这个**"用户主动暴露的意图数据池"**，而不是流量本身。 |

## 5. 创新点

### 5.1 竞价/拍卖/计费

| 点 | 位置 | 谁负责预估 | 备注 |
|---|---|---|---|
| 竞价点 | **意图槽位识别时刻**（m 之前） | 平台（LLM 识别意图） | 新的库存单元 |
| 计费点 | 商品卡露出（m） | 平台直接观测 | 不需要赔付（完全可控）|
| 出价点 | **意图达成事件**（加购 / 留资 / 主动询价） | 平台预估 p(露出→意图达成) | 类似 oCPA |
| 加强点 | 7 日成交额 / 30 日复购 | 平台用 adj(x) 微调 | 类似 Facebook AEO |
| 考核点 | LTV × 用户信任分 | 商家 + 平台双方 | 双目标 |

### 5.2 是否影响自然结果

按照打分判断是否影响，主要可以影响：**候选集、排序、内容权重**。

### 5.3 行业差异：为什么线索行业想象力更强

不同行业差异极大，**重点在线索行业和电商**。其实**线索行业想象力更强，App 下载类更弱**。

**品类匹配度**

线索行业（教育、医美、装修、汽车、法律、保险、医疗咨询、家政）的共性是：

> "决策周期长 + 信息严重不对称 + 用户渴求真实建议 + 单条线索价值高"

这 4 个特征精确命中 AI 对话的能力曲线。

**商业模式优势**

线索行业按 CPL / CPA 计费，LLM 能直接介入"询问 — 筛选 — 促成留资"的整个漏斗，不像电商需要打通商品库、价格、库存、履约一长串后台。技术上线索类 AI Agent 的 **MVP 成本远低于电商**。

**竞争格局**

电商赛道淘宝 / 京东 / 拼多多 / 抖音电商已经卷死，线索行业目前还是百度 SEM 和信息流广告主导，AI 对话天然就是更好的载体——这是**字节弯道超车的稀缺机会**。

**风险点（必须主动提）**

医疗 / 教育 / 金融类线索的合规和虚假宣传风险远高于电商。Agent 给医美建议、给教育做承诺一旦出问题就是品牌灾难。所以起步最佳品类是 **装修、家政、汽车后市场** 这类合规风险中等、决策密度大的领域。

## 6. 架构

<img width="448" alt="image" src="https://github.com/user-attachments/assets/be4e7e36-8b7b-4d39-94ce-c0ac70c68c90" />

> 以下以**电商行业**为例

### 6.1 商家经营

- **一般商家**：结构化商品、SOP、商家人设
- **头部商家**：MCP（思路有点类似于 RTA）

### 6.2 拍卖方式

参考腾讯导购的奖励函数：

```
score = 商业价值 + 意图满足度 − 信任风险
```

其中：

- **商业价值**：CPC / CPS × 预估转化率 × bid（冷启动期需考虑 LTV 项做平滑）
- **意图满足度**：可由语义相似度 / LLM-as-judge / RLHF reward model 综合给出
- **信任风险**：拆成 3 个子维度——**商家历史投诉率 + 商品类目敏感度（医疗 / 金融加权） + 当前会话用户犹豫信号**（停留时间、追问深度）
- **三项权重应动态调整**：决策早期信任风险权重最高，决策末期商业价值权重最高

### 6.3 Agent 编排

- **顶层 Router Agent**：做场景分发
  - **咨询场景** → **Supervisor**（商品 / 优惠 / 对比 / 售后专家）
    - 适配原因：用户意图相对单一明确，Supervisor 一次性分派最合适专家，效率最高
  - **导购场景** → **Swarm**（允许专家间 handoff）
    - 适配原因：意图模糊且会演化，需要多专家自由切换探索
  - **售后场景** → **Pipeline**（固定流程）
    - 适配原因：强流程性 + 强合规要求 + 高确定性，避免 LLM 在合规节点自由发挥

### 6.4 C 端呈现

文本 + 图片 + 富媒体，核心是给用户 **控制感** 和 **信任感**。

## 7. 评估体系

### 7.1 总览

| # | 维度 | 说明 |
|---|------|------|
| 1 | 基本集合 | 测试 session，覆盖所有可能性（意图） |
| 2 | 评估维度 | 准确、相关、一致性、专业度、分寸感？ |
| 3 | 常规评估 | 模型评估 + 人工评估 |
| 4 | 在线评估 | LTV、对话长度轮次、留存、NPS（净推荐值） |

### 7.2 离线 Query 集设计

分 3 类，每类至少 200 条：

- **典型意图**：覆盖头部品类基本流程
- **长尾意图**：小众场景、模糊表达
- **对抗 case**：诱导推荐违禁品、prompt injection、用户钓鱼（"哪款最差"）

### 7.3 标注 rubric

每个维度细化到 **5 档打分细则**，不是"准确性高 / 中 / 低"，而是：

- 5 = 事实 0 错误
- 4 = 1 处不影响决策的小错
- 2 = 1 处影响决策的错

### 7.4 自动指标

- 事实精度（fact precision via grounding check）
- 幻觉率（hallucination rate per category）
- 推荐相关性（query-product semantic match）
- **缺点提及率**（信任机制指标，详见第 9 章）

### 7.5 模型评测

用 Claude Opus / GPT-5 等更强模型当裁判模型对 Agent 输出做盲评，**相关性 ≥ 0.7 才算可靠**。

### 7.6 在线指标分两层

- **北极星**：GMV、对话→下单转化、留存
- **护栏**：客诉率、退货率、单次会话长度异常监控、**商家品牌词搜索量**（后者是检测"AI 是否伤害了品牌建设"的关键反向指标）

## 8. 信任机制设计（如何让用户信任 AI）

### 8.1 代理人信任原则

> AI 推荐的商品必须是在 **满足用户需求的前提下做商业化**，而不是为了商业化去扭曲用户需求的满足。
> **先帮用户选对，在选对的范围内优先展示付费商家。**

### 8.2 三大机制

- **分 AB 区**（详见第 9.2 节讨论）
- **推荐理由 — 可验证**（每条产品声明都能追溯到源头：官方参数页、第三方评测、真实评论）
- **暴露缺点**（主动列出商品的不适配点，是建立信任的核心动作）

### 8.3 防幻觉的产品逻辑

事实性高精度需求（价格、库存、参数、优惠）和 LLM 幻觉天然冲突，必须靠产品架构兜底：

- **双轨架构**：事实性信息强制走结构化商品 DB，LLM 只做表达和适配解释
- **Grounding + Citation**：每一条产品声明可点击追溯出处
- **下单环节硬隔离**：订单生成走传统电商 API，LLM 不参与最后一步
- **评测里专门设计幻觉率指标**：每品类、每意图分桶统计，作为模型上线门槛

## 9. 关键讨论问题

### 9.1 待讨论问题清单

- 是否要分 AB 区？
- 是否要披露当前是广告？
- 是否要披露当前的 Agent 是商家 Agent？

### 9.2 我的判断与解答

**是否分 AB 区**

> 建议**第一阶段不分**，让 AI 回答里直接嵌入商品卡。但**必须用视觉差异化**（背景色 / 边框 / 小标签）让用户能识别。
> 分 AB 区会让用户像看 Feed 广告一样直接划走，违背 AI 导购"理性决策载体"的本质。

**是否披露广告**

> **必须披露，但用"推荐理由"代替"广告"**。比如商品卡上写"基于你的需求匹配（合作品牌）"，而不是写"广告"。
> **披露 ≠ 自损**，反而是建立信任的核心动作。

**是否披露 Agent 是商家 Agent**

> **必须披露**，但同时给用户 **"切换到平台中立 Agent"的选项**。
> 这个设计同时满足了商家诉求（让 Agent 为我代言）和用户诉求（我可以看到中立第三方意见）。

### 9.3 自我蚕食（Cannibalization）


> "AI 导购变现如果做起来，会不会吃掉 Feed 广告和搜索广告的预算？"
> AI 导购吃的是 **理性决策类广告主的搜索预算**（百度 SEM 那部分），**不是 Feed 的兴趣激发预算**。
> 长期看是把外部预算搬进来，不是切自家蛋糕。
> 内部归因机制可以设计成 **"AI 导购 + Feed 协同归因"** —— 如果用户先在 Feed 看到种草、再到 AI 导购完成决策，两边按贡献分账，避免内部内耗。

## 10. AI + 广告能做什么（全链路）

### 10.1 大盘全景

帮助投放的广告中做**创意素材**、帮助做**智能投放**、再就是 **AI 搜 / AI 助手的流量变现**。

### 10.2 全链路环节

> 受众洞察（理解用户）→ 创意生成（做素材）→ 智能投放（选人 + 出价）→ 动态个性化（千人千面）→ 效果归因（衡量结果）→ 新场景变现（AI 搜索 / 助手）

每个环节 AI 都有切入点：

- **受众洞察**：用户行为 + 对话日志的多模态画像
- **创意生成**：即创、Symphony、AIGC 视频 / 图文 / 脚本
- **智能投放**：自动定向 + 自动出价 + 自动预算分配
- **动态个性化**：千人千面落地页 / 千人千面对话开场白
- **效果归因**：跨场景多触点归因
- **新场景变现**：AI 搜 / AI 助手（本项目核心）

## 12. 能力边界

需要主动思考 LLM / Agent 的能力边界，避免做出超过模型能力的产品承诺：

- **上下文信息**：长上下文窗口下的真实可用信息密度
- **单次任务 vs 复杂任务**：何时该让 Agent 自动闭环、何时必须让用户接管
- **结果校验（别的渠道）**：关键事实必须有第二信源验证（结构化数据库 / 第三方权威 API）

## 目前竞价广告的出价框架
框架的灵魂四点（竞价点 / 计费点 / 出价点 / 考核点）是同一条用户漏斗上的四个不同位置，三者关键约束：
竞价点永远在最前端的 m（无论何种出价模式，最终都换算成 eCPM 排序，平台收益最大化）
计费点不能超过平台可完全观测的行为（否则商家会瞒报数据薅羊毛）
出价点越靠后端越接近商家真实价值，但前提是平台对 p(计费点→出价点) 的预估比商家自己的固定值预估更准
一旦出价点和计费点分离（即 oCPX），平台在预估 p(计费点→出价点) 上有"短期偏高"的动机，所以国内必须配 超成本赔付 来重建信任
