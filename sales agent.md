
# AI 营销 Agent 产品思考

## 1. 背景

近期 AI 产品高速发展，目前可见的明显有流量增长的产品形态有两种：

- **对话类**（以豆包为代表）
- **AI 搜**（以 Google AI Overview 为代表）

市场上普遍认为 2026 年为 AI 变现元年，但对于 AI 两种形态的具体变现方式大部分比较固化。根据字节提出的营销 Agent 概念，初步梳理如下。

## 3. 营销 Agent 的概念

> 不确定目前字节营销 Agent 落地程度如何，以下为个人粗略思考。

**产品能力**

可以实现在多轮对话中进行"服务"的推荐和"品牌"的露出，尤其是可以丝滑地在多轮对话中逐步承接用户后端需求。

**变现能力（S 曲线）**

| 阶段 | 判断 | 原因 |
|------|------|------|
| 初期 | 低 | 流量池小（对比 Feed 时长可能短）；曝光机会少（对比搜索，不是单屏幕 5 次）；广告主接受度初期低；技术基建需要时间；用户使用习惯需要培养 |
| 后期 | 高 | 减少交互摩擦，转化率可提升；长尾需求承接；数据量大；预算侧粘度提升 |

## 5. 架构
<img width="448" alt="image" src="https://github.com/user-attachments/assets/be4e7e36-8b7b-4d39-94ce-c0ac70c68c90" />


> 以下以**电商行业**为例

### 5.1 商家经营

- **一般商家**：结构化商品、SOP、商家人设
- **头部商家**：MCP（思路有点类似于 RTA）

### 5.2 排序方式

参考腾讯导购的奖励函数：

```
score = 商业价值 + 意图满足度 - 信任风险
```

### 5.3 Agent 编排

- **顶层 Router Agent**：做场景分发
  - **咨询场景** → Supervisor（商品 / 优惠 / 对比 / 售后专家）
  - **导购场景** → Swarm（允许专家间 handoff）
  - **售后场景** → Pipeline（固定流程）

### 5.4 C 端呈现

文本 + 图片 + 富媒体，核心是给用户**控制感**和**信任感**。

待讨论问题：
- 是否要分 AB 区？
- 是否要披露当前是广告？
- 是否要披露当前的 Agent 是商家 Agent？
